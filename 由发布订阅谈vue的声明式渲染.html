由发布订阅谈vue的声明式渲染 ================= ###### (2018/8/17) 上一篇文章给大家谈了发布订阅模式，相信大家对这个模式有了基本认识。这篇文章我希望能以这个模式为基础，去扩展一个vue的声明式渲染的雏形。如果还不是很懂发布订阅可以先看看这一篇——[简单说说发布订阅](简单说说发布订阅.html)。 那我们就开始了！！！ >[先看vue官网的例子](https://cn.vuejs.org/v2/guide/#%E5%A3%B0%E6%98%8E%E5%BC%8F%E6%B8%B2%E6%9F%93.html) ``` < div id="app"> {{ message }} < /div> var app = new Vue({ el: '#app', data: { message: 'Hello Vue!' } }) ``` >本人理解vue的声明式渲染就是一个由vm层到view层的单向绑定（如有不对的地方请轻拍），即我改变data中的每个属性，他都会相应的更新到页面中。那么如果我们想要实现一个这样的功能，我们只需要为data中每个key创建一个发布者，然后为每个一个绑定key的dom节点创建一个订阅者，然后将他们一一对应收集起来。当data[key]值改变时进行notify，订阅者update触发响应回调函数更新视图。
